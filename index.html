I built an AI agent that does not forget.

Lucy is a self-hosted personal agent — running 24/7, talking over Telegram, built on the Claude Agent SDK. Most of the work went into the hardest part: memory.

01 / What it is

Not a chatbot. A colleague that runs continuously.

A language model forgets everything the moment a conversation ends. Lucy is the opposite: a long-running agent with persistent memory, the ability to take on work on its own, and a real set of tools to act with — not just to talk.

  • a.It remembers. A five-tier long-term memory system, described below.
  • b.It works. Drop a task spec into a queue and Lucy runs a sprint — plan, build, report — by itself.
  • c.It acts. Real tools (files, web, git, finance) through MCP, behind a safety rail.
  • d.It has a cockpit. A web hub for chat, token & cost accounting, expert routing, and skills.
02 / The memory — and why it exists

The point isn't to store text. It's to give the agent continuity.

Without memory, every chat starts from zero — re-explaining who I am, what we're building, what was already decided. That burns tokens and the agent never grows up. The memory system exists to fix four things:

  • 1.Recall the right thing at the right moment — instead of stuffing everything into context.
  • 2.Hold on to people, preferences, projects, decisions — across every conversation.
  • 3.Forget what's stale, deliberately — outdated facts stop being used, but the history is kept.
  • 4.Keep secrets out — sensitive values are redacted before anything is stored or embedded.
A five-tier brain: short-term episodic, long-term hybrid recall, self-consolidation, and controlled forgetting — running on a 1.9 GB VPS.
P0

Recall prefetch

Each turn, relevant memory is looked up (full-text, FTS5) and threaded into the prompt automatically.

P1

Hybrid vector recall

Keyword search fused with semantic search (sqlite-vec + a 768-dim embedding), combined via reciprocal-rank fusion. Index grows incrementally by checksum.

P2

Episodic memory

Individual conversation turns are stored, so Lucy can recall what was actually said, not just curated facts.

P3

Consolidation

Memories are merged the way Mem0 does it — add / update / delete / no-op, with reflection — gated behind a dry-run before anything is applied.

P4

Bi-temporal forgetting

Zep-style validity windows: an expired fact drops out of recall but stays on file as history; a supersede step keeps the new and retires the old on conflict.

Inspired by published memory research from Mem0, Zep and Letta — then redesigned to fit a small box.

03 / The hand

What turns a chatbot into something that can act — safely.

At boot, a tool registry is loaded into a manifest baked into the system prompt — so the agent knows what it can do without scanning files every turn. Every action passes through a guard: dangerous commands are blocked, tool permissions are granted per-persona, and every outcome is logged back into memory.

  • Tool registry + MCPFiles, web, git, memory, finance — discoverable, permissioned.
  • Skill engineSkills load on demand by keyword, instead of bloating every prompt.
  • Safety hooksPre-tool guard, per-persona permission, post-tool audit trail.
04 / What's actually running

Not a demo. A system that's been live for months.

  • Telegram bridgeTelegram talks to the Claude Agent SDK in-process, kept alive by a process manager.
  • Web hubStreaming chat, a token & cost ledger down to source / model / cache, expert and skill panels.
  • Auto-task & auto-buildA queued spec runs a three-phase sprint — plan, build, report — with a low-cost model lane too.
  • Expert routingQuestions are routed to the right persona — finance, marketing, engineering — by tag and embedding.
  • Cost guardA single source of truth for token spend; every code path accrues to one ledger.
  • Daily brief & trend radarMorning and evening cron jobs compile news and market signals into a report.

LinkedIn — draft

I built “Lucy” — a personal AI agent that actually remembers.

As a technical artist and game dev, I got curious: how useful does an AI agent become if it never forgets? So I self-hosted one on the Claude Agent SDK — running 24/7, reachable over Telegram — and spent most of the effort on the hardest part: memory.

Lucy has a five-tier brain, inspired by research from Mem0, Zep and Letta:

— Hybrid recall: keyword + semantic search (sqlite-vec + 768-dim embeddings)
— Episodic memory that stores individual conversation turns
— Self-consolidation (add / update / delete), Mem0-style
— Controlled, bi-temporal forgetting, Zep-style: stale facts stop being used but stay on file

All of it runs on a 1.9 GB VPS.

Beyond memory, Lucy has a “hand”: a tool registry + MCP to work with files, web and git, a safety rail that blocks dangerous commands, and an engine that takes a task and runs it to a report on its own.

The biggest lesson: what turns a chatbot into a colleague isn’t the model — it’s the memory, the tools, and the safety rail around it.
